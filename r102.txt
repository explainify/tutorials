## Hvorfor bruke bibliotek?

Base R kan det meste, men koden blir fort tung og uoversiktlig. Med dplyr og ggplot2:
- **Lesbarhet:** Koden leses mer som setninger («ta data, filtrer, tell, sorter»).
- **Konsistens:** Samme stil på databehandling og grafer.
- **Færre feil:** Mindre repetisjon og enklere å endre steg for steg.

Vi laster inn tidyverse – en pakke som blant annet inkluderer dplyr (data) og ggplot2 (grafer).

:::r
library(dplyr)    ## Laster dplyr som er et bibliotek som hjelper med dataanalys
library(ggplot2)  ## Laster ggplot2 for figurer
npr <- read.csv("https://raw.githubusercontent.com/hmelberg/health_data/refs/heads/main/npr.csv")
head(npr)  ## Viser de 6 første radene
:::

## 3) Lage nye variabler med mutate

::: write
Liggetid finnes ikke i datasettet. Vi lager den fra inn- og utdato, akkurat som i base R – men med mutate() og flere variabler i samme steg.

:::r(silent=true)
npr <- npr %>%
  mutate(
    INNDATO = as.Date(INNDATO),
    UTDATO = as.Date(UTDATO),
    ALDER_BEREGNET = as.integer(format(INNDATO, "%Y")) - FODSELSAAR,
    LIGGETID = as.numeric(UTDATO - INNDATO)
  )
:::

:::r(result_location="below")
npr <- npr %>%
  mutate(
    INNDATO = as.Date(INNDATO),
    UTDATO = as.Date(UTDATO),
    ALDER_BEREGNET = as.integer(format(INNDATO, "%Y")) - FODSELSAAR,
    LIGGETID = as.numeric(UTDATO - INNDATO)
  )
npr %>% summarise(
  min = min(LIGGETID), q1 = quantile(LIGGETID, 0.25),
  median = median(LIGGETID), snitt = mean(LIGGETID),
  q3 = quantile(LIGGETID, 0.75), max = max(LIGGETID)
)
:::

::: write(effect="fade")
mutate() legger til eller overskriver kolonner. Vi skriver bare variabelnavn uten npr$. Flere kolonner lages i ett kall.

::: write_speak
Dette er typisk «tidy»-stil: ett datasett, ett steg per linje, og navnene følger med gjennom pipen.


## 4) Histogram med ggplot2

:::r
npr %>%
  ggplot(aes(x = ALDER_BEREGNET)) +
  geom_histogram(fill = "skyblue", col = "white", bins = 30) +
  labs(
    title = "Aldersfordeling ved innleggelse",
    x = "Alder",
    y = "Antall"
  )
:::

::: write(effect="fade")
Med ggplot2 bygger du grafen lag for lag: først data og aes() (hva er x/y), deretter geom_histogram(), deretter labs() for tekster. Legg til flere lag med +.

::: write_speak
Sammenlignet med hist() i base R gir ggplot2 en mer enhetlig stil og enklere å justere titler, farger og tema senere.


## 5) Telle og vise topp 10 prosedyrer

:::r
top10 <- npr %>%
  count(PROSEDYRE1, sort = TRUE) %>%
  slice_head(n = 10)
top10
:::

::: write(effect="fade")
count() teller forekomster per gruppe; sort = TRUE gir synkende rekkefølge. slice_head(n = 10) tar de 10 øverste radene. Ingen table(), order() eller as.data.frame().

:::r
top10 %>%
  ggplot(aes(x = n, y = reorder(PROSEDYRE1, n))) +
  geom_col(fill = "steelblue") +
  labs(title = "Topp 10 prosedyrer", x = "Antall", y = "Prosedyrekode")
:::

::: write(effect="fade")
reorder(PROSEDYRE1, n) sorterer kategoriene på y-aksen etter antall. geom_col() er søylediagram når du allerede har tellinger (geom_bar() teller selv).

::: write_speak
Her ser du typisk dplyr + ggplot2: først rydde og telle med count() og slice_head(), deretter pipe direkte inn i ggplot(). Samme data, ingen ekstra variabler.


::: write_speak
## Oppsummering: base R vs. bibliotek

Med base R bruker du read.csv(), table(), order(), hist() og barplot(). Det fungerer fint. Med dplyr og ggplot2 bruker du read.csv() eller read_csv() til innlasting, deretter count(), mutate(), ggplot() og geom_*(). Koden blir ofte kortere og mer lesbar, og du får en konsistent måte å jobbe på – spesielt når datasettet vokser eller du skal dele kode med andre. Begge veier er gyldig; bibliotekene er verktøy for å jobbe raskere og tydeligere.
